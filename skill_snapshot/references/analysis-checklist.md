# 数据科学分析检查清单

在高风险分析、建模、实验分析或数据集含义不清时使用这个清单。

## 数据质量

- 确认每一行的粒度：用户、交易、事件、账户、日期，还是其他对象。
- join 之前检查主键和重复率。
- 按字段、关键分组检查缺失情况。
- 核对数值单位、货币、百分比、日期时区，以及时间边界是否包含端点。
- 过滤或 join 前后都检查行数、汇总值和日期范围。
- 检查可疑常数、不可能值、负值和极端异常值。

## 指标定义

- 明确分子、分母、分析对象、时间窗口和排除规则。
- 判断指标应该加权、按行平均、按实体平均，还是先聚合再平均。
- 区分领先指标和结果指标。
- 分析留存、转化、流失或长期行为时，不要混淆不同 cohort。

## 实验和因果判断

- 检查分流单位、处理发生时间、真实曝光和污染风险。
- 报告样本量、基线平衡、效应量、置信区间、必要时报告 p 值，以及实际业务意义。
- 谨慎使用前后对比；必要时说明季节性、选择偏差、均值回归和外部冲击。
- 只有研究设计支持因果推断时，才使用因果语言。

## 建模

- 特征工程前先定义目标变量、预测窗口、预测单位和评估指标。
- 按部署方式切分数据：有时间泄漏风险时用时间切分，实体重复出现时考虑 group 切分。
- 先建立简单 baseline。
- 检查来自未来字段、结果后字段、ID、target encoding 和聚合特征的数据泄漏。
- 在留出集上评估，并按关键分组检查错误。
- 说明模型限制和上线假设。

## 沟通

- 先给答案，再给证据。
- 区分事实、假设、解释和建议。
- 说明哪些 caveat 可能改变决策。
- 表格和图表要有清楚的轴名、单位和日期范围。
