# Data Science

## 工作原则

把每个数据任务都当成一个“建立证据”的过程。优先使用可复现代码、明确假设和简洁结论，不做临时拼凑式计算。分析要紧贴用户真正的问题，但也要充分检查数据，避免被字段含义、数据泄漏、缺失值、重复记录、异常值和单位不一致误导。

不要编造数据、字段、指标定义或业务背景。缺少必要定义时，低风险场景可以说明假设后继续推进；如果会明显影响结论，就先向用户确认。

## 标准流程

1. 明确问题：确认目标指标、分析粒度、时间窗口、分析对象、排除规则和期望输出。
2. 检查输入：列出文件、字段、行数、数据类型、缺失情况、重复键、样例值、日期范围和类别字段基数。
3. 准备数据：清洗类型，规范字段名，谨慎处理缺失值，保留原始输入，并记录过滤和 join 逻辑。
4. 开始分析：根据问题选择方法。先做简单描述性检查，再考虑复杂统计或建模。
5. 验证结果：检查 join 前后行数和汇总值，能对照已知基准时就对照，检查边界情况，并留意泄漏和混杂因素。
6. 表达结论：先给答案，再给证据；说明不确定性和 caveat；保留可复现命令或输出文件；不要把相关性夸成因果性。

## 工具默认选择

优先使用当前项目已有的技术栈。独立分析任务默认使用 Python，常用库包括 pandas、numpy、scipy、statsmodels、sklearn、matplotlib 或 seaborn。只有用户明确需要探索式 notebook 时才优先创建 notebook；需要复跑时优先写脚本。

如果任务涉及 Excel 格式、公式、多 sheet 工作簿编辑、格式或图表对象，配合 spreadsheet 相关 skill 使用；本 skill 主要负责数据分析思路和判断。

快速查看 CSV、TSV、Excel、JSON 或 Parquet 文件时，使用：

```bash
python scripts/profile_table.py path/to/data.csv --out profile.json
```

当任务涉及建模、因果判断、实验分析、数据质量排查或重要决策时，阅读 `references/analysis-checklist.md`。

## 常见分析模式

探索性分析：先生成简洁的数据概览，再围绕用户问题做分组汇总、分布、趋势和适合画图的表。优先选择少量清晰图表，不堆叠装饰性图。

统计推断：说明假设、样本量、检验选择、前提条件、效应量、置信区间和实际意义。参数检验前提较弱时，说明限制，并考虑稳健或非参数检查。

建模任务：先建立 baseline，再按部署场景选择切分方式，训练前明确目标变量和评估指标，公平比较模型，检查错误样本，并解释取舍。不要在 baseline 还没测过时直接上复杂模型。

时间序列：先排序和检查时间戳，确认缺口和聚合粒度；预测任务避免随机切分；区分趋势、季节性、干预事件和异常值。

面向汇报的分析：输出结构要清楚，包括指标定义、核心结论、支撑表格或图表、以及可能影响判断的限制。

## 交付物

除非用户指定，优先使用这些文件名：

- `analysis.py` 或 `notebook.ipynb`：可复现分析
- `profile.json` 或 `profile.md`：数据概览
- `cleaned_<name>.csv`：清洗后的表格数据
- `figures/`：导出的图表
- `findings.md`：最终分析结论

最终回复里说明：分析了什么、创建或修改了什么、核心结论是什么、做过哪些验证、还有哪些 caveat。
